# Freshmall 数据库表说明

本文基于 `doc/sql/freshmall_db_backup_2026-03-16.sql`。

## 一、核心业务表

| 表名 | 用途 | 说明 |
| --- | --- | --- |
| `b_user` | 用户主表 | 用户账号、密码、基础资料、角色等。 |
| `b_thing` | 商品表 | 农产品基础信息、价格、库存、分类等。 |
| `b_classification` | 分类表 | 商品分类定义。 |
| `b_cart` | 购物车表 | 用户购物车条目，支持多条记录。 |
| `b_order` | 订单表 | 订单主体，包含状态、收货信息、`payment_no` 聚合支付号。 |
| `b_thing_collect` | 收藏表 | 用户收藏商品关系。 |
| `b_thing_wish` | 点赞表 | 用户点赞商品关系。 |
| `b_record` | 行为记录表 | 用户浏览/行为记录（用于推荐等）。 |

## 二、订单事件与库存一致性表

| 表名 | 用途 | 说明 |
| --- | --- | --- |
| `b_order_event_outbox` | Outbox 事件表 | 订单服务本地事务写入的待投递事件。 |
| `b_order_event_process_log` | 事件处理日志表 | 事件投递/处理状态跟踪，支撑重试和幂等。 |
| `b_stock_event_consume_log` | 库存消费日志表 | 商品服务消费库存事件的幂等日志。 |

## 三、运维与日志表

| 表名 | 用途 | 说明 |
| --- | --- | --- |
| `b_login_log` | 登录日志 | 登录行为记录。 |
| `b_op_log` | 操作日志 | 后台操作行为记录。 |
| `b_error_log` | 错误日志 | 系统异常记录。 |

## 四、通知相关表（建议暂时下线）

| 表名 | 当前状态 | 建议 |
| --- | --- | --- |
| `b_notice` | 系统通知/公告数据 | 当前前后端通知功能已移除，可作为待删除表。 |

建议先观察一段时间（确认无外部依赖任务）再执行删除：

```sql
-- 建议在备份后执行
DROP TABLE IF EXISTS `b_notice`;
```

## 五、本次“通知功能下线”对应代码侧变更说明

- 客户端移除：头部通知抽屉、个人中心“系统通知”入口、`messageView` 路由。
- 管理端移除：通知公告菜单、`notice` 路由、公告管理页面。
- 后端移除：`Notice` 实体类（`b_notice` 的代码映射）。

以上变更仅下线通知功能，不影响订单支付、购物车、商品与用户主流程。
